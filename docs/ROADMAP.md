Roadmap
Vision générale

Construire progressivement un mini moteur web spécialisé capable de supporter :

navigation fluide

plusieurs vues caméra

monde extensible

construction simple

missions minimales

assets légers

direction artistique cohérente

Phase 1 — Refactorisation V0.1

Objectif : stabiliser la base technique actuelle.

En cours

v0.1a : code sorti du HTML

v0.1b : config / state / utils / input / HUD

v0.1c : renderer + camera controller

v0.1d : world (chunks / collision / walls)

À faire

v0.1e : entities (player / enemies / coins)

v0.1f : FX + particules

v0.1g : game loop + reset + nettoyage final

Phase 2 — Base visuelle et sensorielle V0.2

Objectif : enrichir sans alourdir.

Cibles

petite librairie de meshes low poly

textures légères

premiers sons

petits FX graphiques

premiers props décoratifs

monde plus vivant visuellement

Phase 3 — Exploration structurée V0.3

Objectif : renforcer l’identité du prototype.

Cibles

mode exploration plus affirmé

meilleure lecture spatiale

outils simples proches d’un mini éditeur

meilleure séparation mission / exploration

éventuel zoom unique

Phase 4 — Systèmes de jeu V0.4

Objectif : enrichir le contenu sans casser la simplicité.

Cibles

missions variées

règles de monde plus riches

nouveaux types d’interactions

premières logiques de progression

Phase 5 — Évaluation stratégique

Objectif : décider lucidement de la suite.

Questions

le mini moteur reste-t-il assez fluide ?

la maintenance reste-t-elle simple ?

l’ajout de features reste-t-il raisonnable ?

faut-il continuer en web ?

faut-il préparer une migration vers Unity ?

Règle de progression

Ne pas ajouter de systèmes lourds trop tôt.

Priorité constante :

clarté

stabilité

fluidité

modularité

évolution progressive
